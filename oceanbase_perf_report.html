分析概述

本报告基于 OSDI 2025 论文《Principles and Methodologies for Serial Performance Optimization》的三大原则（Task Removal、Task Replacement、Task Reordering）和八大可操作方法论，对 OceanBase 分布式数据库进行了全面的性能优化分析。

关键发现：OceanBase 的 LSM-Tree 存储引擎、分布式事务处理（Paxos-based 2PC）和 SQL 执行引擎存在显著优化空间。通过系统性地应用 OSDI 2025 方法论，预计可实现 25-40% 的端到端性能提升。

1

Compaction 优化、缓存策略、批量写入

3

计划缓存、向量化执行、并行下推

RPC 批量化、数据本地化、网络优化
✅ 当前优势
  • LSM-Tree 架构天然支持高吞吐写入
  • MVCC + 悲观锁混合并发控制
  • Paxos 协议保证强一致性
  • 多层级缓存（Row Cache、Block Cache、Bloom Filter）
  • 向量化执行引擎
  • 共享无架构（Shared-nothing）支持线性扩展
⚠️ 潜在瓶颈
  • Compaction 期间的性能抖动
  • GTS 可能成为分布式事务瓶颈
  • 跨节点查询的网络开销
  • MemTable 内存压力
  • 大量小查询的计划生成开销
  • Row Cache 命中率波动

方法论应用分析

📐 三大核心原则

🗑️ Task Removal（任务移除）

识别并消除不必要的工作。在 OceanBase 中应用：

  • 移除冗余的日志复制
  • 消除不必要的缓存查找
  • 跳过无意义的版本链遍历

🔄 Task Replacement（任务替换）

用更高效的实现替换低效操作。在 OceanBase 中应用：

  • 替换串行 GTS 获取为批量获取
  • 用自适应算法替换固定策略
  • 用向量化操作替换逐行处理

📋 Task Reordering（任务重排序）

调整执行顺序以减少依赖或提高并行性。在 OceanBase 中应用：

  • 重排序 Compaction 任务
  • 调整 Paxos 复制顺序
  • 优化 2PC 阶段顺序

八大可操作方法论详解

Batching 批处理优化

▼

核心思想：合并多个小操作成大操作，减少开销（系统调用、网络往返、锁获取）

OceanBase 应用场景

应用点 当前实现 优化潜力
GTS 获取 每个事务单独请求 GTS 高 - 可批量获取
Paxos 日志复制 Group Commit 已部分实现 中 - 可进一步优化批大小
RPC 调用 单次 RPC 较多 高 - 合并跨节点请求
索引构建 逐行插入 高 - 批量宏块写入
预期收益：通过 Batching 优化，预计可减少 30-50% 的网络往返和系统调用开销，特别是在高并发短事务场景下。

Caching 缓存优化

▼

核心思想：缓存计算结果避免重复计算

当前缓存层级

  • Row Cache: 单行查询结果缓存
  • Block Cache: 数据块缓存
  • Bloom Filter Cache: 空查过滤缓存
  • Plan Cache: 执行计划缓存

优化机会

问题识别：Row Cache 命中率波动，版本链过长时缓存失效频繁
  • 引入自适应缓存替换策略（ARC/LIRS 替代 LRU）
  • 多级缓存一致性优化
  • 预取策略优化（基于访问模式预测）
  • 缓存预热优化（Compaction 期间预加载）

Precomputing 预计算优化

▼

核心思想：提前计算可能需要的值

OceanBase 应用机会

预计算点 说明 收益
Bloom Filter Compaction 时预计算 SSTable 层 Bloom Filter 减少点查磁盘 I/O
统计信息 后台预计算表/列统计信息 优化器选择更优计划
物化视图 预计算聚合结果 OLAP 查询加速
分区裁剪 预计算分区范围 减少扫描分区数

Deferring 延迟执行优化

▼

核心思想：延迟非紧急工作到更合适时机

OceanBase 已有实践

  • 增量 Major Compaction: 延迟到业务低峰期执行
  • DDL 延迟执行: 添加列等 DDL 延迟到 Compaction 时处理

进一步优化机会

  • 异步索引构建: 进一步延迟非关键索引的构建
  • 延迟统计更新: 小变更批量后统一更新统计
  • 惰性日志刷盘: 非关键事务延迟刷盘（可控一致性降级）
  • 批量垃圾回收: MVCC 版本批量清理而非实时

Relaxation 语义放松优化

▼

核心思想：放宽严格语义以换取性能

潜在优化方向

注意：Relaxation 需要谨慎应用，需评估对业务的影响
  • 读己之写延迟: 允许跨节点读存在一定延迟
  • 单调读放松: 从库读取允许一定程度的版本滞后
  • 因果一致性选项: 提供因果一致性读写接口
  • Paxos 复制放松: 非关键数据异步复制

Contextualization 上下文感知优化

▼

核心思想：根据上下文选择最优策略

OceanBase 已有实现

  • 自适应 GROUP BY 下推: 根据采样数据决定是否下推
  • 动态 Bloom Filter: 根据数据特征动态创建

扩展优化机会

  • 自适应 Compaction 策略: 根据负载选择 minor/major compaction 频率
  • 上下文感知缓存: 根据查询模式动态调整缓存策略
  • 自适应锁策略: 低冲突场景使用乐观锁替代悲观锁
  • 负载感知的 GTS 批大小: 根据并发度动态调整

Hardware Specialization 硬件特化优化

▼

核心思想：利用特定硬件特性加速

硬件优化方向

硬件 优化机会 预期收益

Layering 分层架构优化

▼

核心思想：通过分层架构隔离和优化

OceanBase 分层现状

  • 已良好分层: SQL层、事务层、存储层、分布式层
  • 优势: 独立扩展、故障隔离

优化机会

  • 存储格式分层: 热数据行存、冷数据列存
  • 事务隔离层: 短事务快速路径 vs 长事务完整协议
  • 查询执行分层: 简单查询短路 vs 复杂查询优化
  • 网络通信分层: 控制平面与数据平面分离
